# 🎉 MusicPilot v1.0.0 - 发布说明

发布日期：2026-02-27

---

## 📌 版本信息

**MusicPilot v1.0.0** 是首个正式发布版本，标志着项目从开发阶段进入稳定可用阶段。这个版本完整实现了音乐库自动化管理的核心功能，包括元数据获取、播放列表管理、媒体库管理、资源站点对接、下载器集成、订阅功能和播放器界面。

---

## ✨ 核心特性

### 1. 元数据系统
- 🎵 基于 MusicBrainz 的元数据自动识别和补全
- 🔍 支持搜索艺术家、专辑、曲目
- 💾 智能缓存机制，减少重复请求

### 2. 播放列表管理
- 📝 完整的播放列表 CRUD 操作
- 🎧 曲目拖拽排序
- 🔀 随机播放和循环播放支持

### 3. 媒体库管理
- 📁 自动扫描和整理音乐文件
- 🔄 从下载目录转移到媒体库
- 🏷️ 自动写入元数据标签

### 4. 资源站点和下载器对接
- 🌐 支持多个资源站点（M-Team、Great Discord 等）
- 🔄 自动解析 RSS 种子源
- 📥 集成 qBittorrent 和 Transmission 下载器
- 📊 实时监控下载进度

### 5. 订阅功能
- 🎤 艺术家订阅
- 💿 专辑订阅
- 📋 歌单订阅（网易云音乐、QQ 音乐）
- 🏆 榜单订阅
- 📏 灵活的订阅规则匹配（格式、比特率、大小）

### 6. 播放器界面
- 🎨 基于 Howler.js 的现代化播放器
- ⌨️ 快捷键支持（空格播放/暂停、方向键控制、M 键静音）
- 📊 四种音频可视化效果（频谱、波形、柱状图、圆形）
- 🎤 歌词显示和同步滚动

---

## 🏗️ 技术架构

### 后端
- **框架**：FastAPI 0.115+
- **数据库**：SQLite / PostgreSQL (SQLAlchemy 2.0+)
- **缓存**：内存缓存 / Redis
- **任务调度**：APScheduler
- **代码风格**：Black + Ruff

### 前端
- **框架**：Vue 3.5+
- **UI 组件**：Naive UI 2.40+
- **状态管理**：Pinia 2.3+
- **路由**：Vue Router 4.5+
- **音频播放**：Howler.js 2.2+
- **可视化**：Web Audio API

## 🚀 快速开始

### 使用 Docker（推荐）

```bash
git clone https://github.com/hhyo/MusicPilot.git
cd MusicPilot
docker-compose up -d
```

### 手动部署

#### 后端
```bash
cd backend
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

#### 前端
```bash
cd frontend
npm install
npm run build
```

详细部署文档请参考 [DEPLOY.md](DEPLOY.md)。
